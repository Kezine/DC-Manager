Barre d'outils de la vue — 3D

En vue 3D, la barre porte à GAUCHE le champ de recherche GLOBAL (tout résultat cliqué navigue vers la 3D), à DROITE le contrôle segmenté de mode de vue et le filtre Sites. La popover de résultats est montrée OUVERTE, sortie de son ancrage position:absolute.

.rm-toggle = contrôle segmenté (CHOIX 1 parmi N) : un seul conteneur bordé, segment actif teinté (fond non plein, texte accent). Le mode de vue étant un état PERSISTANT, chaque segment porte aria-pressed (true sur le segment actif) pour les lecteurs d'écran. Le filtre .multi-trigger est montré FERMÉ (son panneau .multi-panel est masqué tant qu'il n'est pas .open).

Barre d'outils de la vue — Plan de salle (2D)

En vue 2D (Dessus / Étage), le champ de recherche disparaît (les résultats forcent la 3D). Apparaissent alors à GAUCHE les bascules d'édition contextuelles (.dc-subviews), à DROITE le segmenté et le filtre Sites.

La bascule « Cases inaccessibles » est montrée ACTIVE (état .active = accent). Les bascules PERSISTANTES (« Placement libre », « Cases inaccessibles ») portent aria-pressed reflétant leur état ; « Éditer la salle » est une ACTION (ouvre un formulaire) et n'en porte pas. En Plan d'étage, « Éditer la salle » devient « Éditer l'étage ».

Bandeau de contrôles 3D (dc-control-bar)

Overlay horizontal en haut du canevas (normalement position:absolute, sorti de son ancrage ici). État figé = vue 3D bureau : zoom, recentrage, points de vue (cube dont la face regardée est liserée d'accent), plein écran, projection, mesure, export. Les boutons contextuels sont conservés dans le DOM mais masqués.

États FIGÉS reproduits du DOM réel : le bouton RETOUR (data-act="back") reste masqué hors historique de navigation ; l'outil POSITIONNEMENT (data-act="position") n'apparaît qu'en 2D ; le bouton « Réglages 3D » (.gz-resp-toggle) est réservé au responsive (masqué sur grand écran par le CSS). La projection est montrée en ORTHOGRAPHIQUE (lignes parallèles) ; en perspective, l'icône passe aux lignes fuyantes.
